## 1. 테스트 재현 명령어 (Reproducibility)

다음 명령어들을 순서대로 실행하여 본 리포트의 결과를 재현할 수 있습니다.

```bash
# 1. 환경 설정 확인
direnv allow
pre-commit run --all-files

# 2. 데이터 수집 (Full Sync)
direnv exec . uv run octopith sync https://github.com/parjong/prototype

# 3. 데이터 수집 (Partial Sync - 시간 제한 검증)
direnv exec . uv run octopith sync https://github.com/pallets/click --duration 5s

# 4. 검색 기능 검증 (JSON 출력 및 위치 구분)
direnv exec . uv run octopith search "synchronization" --fast

# 5. 검색 기능 검증 (Rich Table 출력)
direnv exec . uv run octopith search "synchronization" --fast --rich

# 6. 도움말 시스템 검증 (SSOT 기반)
direnv exec . uv run octopith search --help

# 7. 유닛 테스트 실행
uv run pytest tests/
```

## 2. 주요 기능 검증 결과

### 2.1 수집 및 동기화 (Sync Engine)
- **Full Sync**: `parjong/prototype`의 모든 이슈 및 댓글 수집 완료.
- **Partial Sync**: `pallets/click` 수집 중 5초 도달 시 "Duration reached" 로그와 함께 안전하게 중단됨을 확인.

### 2.2 고도화된 검색 (Agent-Friendly Search)
- **복합 매칭 예제 (Keyword: "synchronization")**:
    - 동일 이슈(#37)에 대해 `THREAD_TITLE`, `THREAD_BODY`, `COMMENT_BODY`가 각각 구분되어 출력됨을 확인.
    - 에이전트가 검색 결과의 맥락(제목인지, 본문인지, 댓글인지)을 정확히 파악 가능.
- **포맷**: 기본 JSON 출력 제공으로 에이전트 자동화 지원.

### 2.3 도움말 시스템 (Dynamic CLI Help)
- **SSOT 적용**: `MatchType` 상수를 기반으로 도움말 메시지 동적 생성.
- **구조화된 안내**: 100컬럼 이내의 가독성 확보 및 `OUTPUT INFORMATION`, `MATCH TYPE DETAILS` 섹션 구분.
- **설계 원칙**: 자세한 내용은 [00.DESIGN.md: CLI 설계 원칙](../00.DESIGN.md#3-cli-설계-원칙-cli-design-principles)을 참조.
