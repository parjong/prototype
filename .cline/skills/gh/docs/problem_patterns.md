# Github CLI 문제 패턴 기록 가이드라인

Github CLI를 사용할 때 피해야 할 문제 패턴과 그에 대한 가이드라인입니다. 각 패턴에 대한 자세한 내용은 다음 링크를 참조하십시오.

### 패턴 설명 구조

각 문제 패턴은 다음과 같은 구조로 작성됩니다.

1.  **패턴 개요**: 문제점, 상세 설명, 그리고 핵심적인 교훈을 포함하는 간결한 서술입니다.
2.  **나쁜 예시**: 문제 패턴을 보여주는 코드 또는 명령어 예시 (잘못된 사용법)
3.  **좋은 예시 (권장)**: 문제 패턴을 개선하거나 피하는 방법을 보여주는 코드 또는 명령어 예시 (권장 사용법)
